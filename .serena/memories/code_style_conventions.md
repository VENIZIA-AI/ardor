# ARDOR Code Style & Conventions

## TypeScript Standards

### Strictness
- Use **strict TypeScript** mode
- Avoid `any` types - use proper typing
- Use explicit return types for public functions
- Prefer interfaces over type aliases for object shapes

### Module System
- ESM modules only (import/export)
- Use barrel exports via auto-generated index.ts files

### Path Aliases
All code uses these path aliases (configured in tsconfig.json):
```typescript
"@/*"           → "src/*"
"@/components"  → "src/components"  
"@/utilities"   → "src/utilities"
"@/hooks"       → "src/hooks"
"@/ui"          → "src/components/shadcn"
```

## File Naming
- React components: PascalCase (e.g., `Button.tsx`, `AdaptiveDialog.tsx`)
- Utilities/hooks: kebab-case (e.g., `use-mobile.ts`, `tw.utility.ts`)
- Test files: `*.test.ts` or `*.test.tsx`
- Type definitions: `*.d.ts`

## Component Structure

### File Organization
```
components/
├── shadcn/          # Auto-generated shadcn components (don't manually edit)
├── core/            # Custom components
│   ├── adaptive/    # Responsive adaptive components
│   ├── common/      # Shared utility components
│   └── backdrop/    # Backdrop/overlay components
└── icons/           # Icon components
```

### Component Patterns
- Use Radix UI primitives as base
- Export components with named exports
- Use cva for variant management
- Compose with forwardRef for ref forwarding
- Use displayName for debugging

## React Patterns
- Functional components only
- Hooks for state management
- Prop destructuring in component signatures
- TypeScript interfaces for props

## Styling Patterns
- Tailwind utility classes
- CSS variables for theming (--background, --foreground, etc.)
- Use cn() utility (tailwind-merge + clsx) for conditional classes
- Follow "new-york" shadcn style variant

## Import Order
1. External libraries (react, etc.)
2. Internal path aliases (@/*)
3. Relative imports (./)
4. Type imports (separate)

## Commit Conventions
Follow Conventional Commits:
- `feat:` - New features
- `fix:` - Bug fixes
- `docs:` - Documentation
- `chore:` - Maintenance tasks
- `refactor:` - Code restructuring
- `test:` - Test additions

## Linting & Formatting
- ESLint config from @venizia/dev-configs
- Prettier with Tailwind class sorting
- Pre-commit hooks enforce linting
- Auto-fix with `bun run lint:fix`

## Special Conventions

### Auto-generated Files
- `src/index.ts` - Auto-generated by generate-index.ts (don't edit manually)
- `src/components/shadcn/*` - Auto-generated by shadcn CLI (can edit after generation)

### Component Exports
All components/hooks/utilities are auto-exported via generate-index.ts which scans:
- `src/components/**/*`
- `src/hooks/**/*`
- `src/utilities/**/*`

Excludes: `*.test.ts`, `*.test.tsx`, `*.d.ts`, `index.ts`
